PROMPT 2 — Fixed-Width Binary Search Index
Question asked to AI:
  "Giải thích cách triển khai binary search trực tiếp trên ổ đĩa (disk-based)
   với fixed-width records 53 byte. Tại sao fixed-width giải quyết được vấn đề
   variable-length records? Viết thuật toán while left <= right."

AI assistance received:
  - Explained the math: seek(mid * 53) lands exactly on record boundary
  - Clarified that O(log n) binary search works only if index is sorted alphabetically
  - Warned about UTF-8 encoding: len(s.encode('utf-8')) != len(s) for Vietnamese text
  - Provided the binary search loop with left/right/mid pattern

Code generated with AI help: index_navigator.py (binary search method)

PROMPT 5 — LRU Cache Integration
Question asked to AI:
  "Tại sao không thể dùng @functools.lru_cache decorator trực tiếp trên
   phương thức của class? Cách giải quyết đúng là gì?"

AI assistance received:
  - Explained: instance methods receive 'self' as first arg, which is not hashable
    by lru_cache when the instance contains mutable state (file handles)
  - Solution: in __init__, cache a bound function reference:
    self._lookup_cached = lru_cache(maxsize=256)(self._lookup)
  - This makes the cache specific to the instance, not the class

Code incorporated into: app.py
